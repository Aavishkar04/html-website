SECTION 1

The Math You Need — And Why Exactly This Math

 

This section answers one question precisely: which math topics directly pay you back in DSA, competitive programming, and backend engineering. Nothing extra. Nothing that belongs in a data science or ML track.

 

The Core Rule

Every math topic in this document maps to a specific DSA problem type or coding concept. If it doesn't map, it's not in here. This is not a math course — it's the math subset that makes you faster and smarter at coding problems.

 

1.1  Arithmetic & Number Theory  (Month 1)

This is where you start. You already know basic arithmetic — but competitive programming uses number theory constantly. Here is what you need to understand deeply:

 

Modulo Arithmetic

You already used % today without understanding it. Now own it completely.

•       n % 2 == 0 means n is even because dividing by 2 leaves no remainder

•       n % 10 gives you the last digit of any number — used in digit-based problems

•       (a + b) % m == ((a % m) + (b % m)) % m — this property appears in hundreds of LeetCode problems

•       Use case: checking divisibility, hashing, cyclic patterns, clock problems

 

Khan Academy link: search 'Remainders' and 'Modular arithmetic' — complete both modules

 

Prime Numbers & Sieve of Eratosthenes

A prime is a number greater than 1 with no divisors except 1 and itself. You need to:

•       Know how to check if n is prime using a loop up to sqrt(n) — not up to n. This alone reduces time complexity from O(n) to O(sqrt n)

•       Know the Sieve of Eratosthenes — finds all primes up to N in O(N log log N). Appears in competitive programming constantly

•       Use case: prime factorization problems, GCD/LCM, cryptography-adjacent problems

 

GCD and LCM

•       GCD (Greatest Common Divisor) — use Euclidean algorithm: gcd(a, b) = gcd(b, a % b)

•       LCM (Lowest Common Multiple) = (a * b) / gcd(a, b)

•       These appear in fraction simplification, interval problems, tile problems

•       Java built-in: no native gcd — you write it yourself in competitive programming

 

Powers and Logarithms

This is the single most important math topic for understanding Big O notation.

•       2^10 = 1024 — memorize powers of 2 up to 2^20. Essential for binary problems

•       log2(n) tells you how many times you can halve n before reaching 1 — that's why binary search is O(log n)

•       log10(n) + 1 tells you how many digits n has — used in digit problems

•       If an array doubles in size each step, that's exponential growth — O(2^n). Recognize this pattern in recursion

 

Khan Academy: complete 'Introduction to logarithms' and 'Logarithm properties'. Do every exercise, not just the videos.

 

 

 

1.2  Algebra & Functions  (Month 1-2)

Not textbook algebra — the algebra patterns that appear constantly in coding:

 

Variables and Equations as Code

•       x = y + 5 in algebra is identical to int x = y + 5 in Java. The mental model is the same

•       Solving for unknowns in a formula is exactly what you do when computing array indices, loop bounds, time complexity

•       Practice: Khan Academy Algebra 1 — focus on linear equations and inequalities

 

Functions and Mapping

•       f(x) in math is identical to a method in Java. Input goes in, output comes out

•       Composition f(g(x)) is the same as calling a method inside another method

•       This mental model makes recursion intuitive — a function calling itself is just f(f(x))

 

Sequences and Series

•       Arithmetic sequence: 1, 3, 5, 7... increases by constant. Sum formula = n/2 * (first + last)

•       Geometric sequence: 1, 2, 4, 8... multiplies by constant. This is exponential growth

•       These appear in: array prefix sums, time complexity of loops, dynamic programming

 

 

 

1.3  Discrete Mathematics  (Month 2-3)

This is the most important math section for competitive programming. Discrete math is the math of countable things — exactly what computers work with.

 

Logic and Boolean Algebra

•       AND (&&), OR (||), NOT (!) — you use these in code every day without knowing the formal system

•       Truth tables: given inputs, what is the output? Practice building them

•       De Morgan's Law: NOT(A AND B) = NOT(A) OR NOT(B) — appears in simplifying conditional logic in code

•       XOR: a ^ b = 1 when bits differ, 0 when same. Used in bit manipulation problems constantly

 

Sets

•       A set is a collection of unique elements — like a HashSet in Java

•       Union, intersection, difference — these map directly to set operations in code and SQL

•       Use case: finding common elements between arrays, duplicate detection, graph connectivity

 

Combinatorics — Counting Problems

This is critical for competitive programming and appears in LeetCode medium/hard problems:

•       Permutations: how many ways to arrange n items? Answer: n! (n factorial). 5! = 5*4*3*2*1 = 120

•       Combinations: how many ways to choose k items from n? Answer: C(n,k) = n! / (k! * (n-k)!)

•       These appear in: path counting, subset problems, dynamic programming

•       Pascal's Triangle: a visual way to compute combinations. Memorize the first 7 rows

 

Pigeonhole Principle

•       If you have n+1 items in n boxes, at least one box has 2 items

•       Sounds trivial. Appears in: duplicate detection, birthday problem, hash collision proofs

•       Example: in any array of 367 people, at least 2 share a birthday

 

 

 

1.4  Recursion Math  (Month 3)

Recursion is both a coding pattern and a mathematical concept. Understanding the math makes the code obvious:

 

Recurrence Relations

•       Fibonacci: F(n) = F(n-1) + F(n-2). The value depends on previous values — same as recursive code

•       Every recursive function has a recurrence relation. Writing the math first makes the code easier

•       Base case in math = base case in code. No base case = infinite loop in both

 

Mathematical Induction

•       Prove something works for n=1, then prove if it works for n it works for n+1

•       This is exactly how you verify that a recursive function is correct

•       Not required for interviews but builds deep intuition for why recursion works

 

 

 

1.5  Probability Basics  (Month 4)

Only the basics. Not statistics. Just enough for DP problems involving probability and interview questions:

 

•       Probability of event A = (favorable outcomes) / (total outcomes)

•       P(A and B) = P(A) * P(B) when A and B are independent

•       P(A or B) = P(A) + P(B) - P(A and B)

•       Expected value: sum of (outcome * probability of outcome) — appears in game theory problems

•       Khan Academy: complete 'Basic probability' and 'Probability with counting' modules

2.2  Java Syntax You Must Know Before Starting DSA

Before writing a single DSA solution, these Java concepts must be automatic — no lookup needed:

 

Data Types and Variables

•       int (32-bit), long (64-bit, for large numbers), double (decimals), char, boolean, String

•       int vs long: whenever a problem says 'numbers up to 10^18', use long not int — int overflows

•       Integer.MAX_VALUE = 2147483647. Long.MAX_VALUE = ~9.2 * 10^18. Know these

 

Arrays

•       int[] arr = new int[5]; — fixed size, zero-indexed

•       int[] arr = {1, 2, 3, 4, 5}; — initialized array

•       arr.length — not arr.length(), no parentheses for arrays

•       Arrays.sort(arr) — sorts in ascending order. Know this exists

•       Arrays.fill(arr, 0) — fills array with a value

 

Java Collections — The Four You Must Know

•       ArrayList<Integer> list = new ArrayList<>(); — dynamic array. list.add(), list.get(i), list.size(), list.remove(i)

•       HashMap<String, Integer> map = new HashMap<>(); — key-value store. map.put(), map.get(), map.containsKey(), map.getOrDefault(key, 0)

•       Stack<Integer> stack = new Stack<>(); — LIFO. stack.push(), stack.pop(), stack.peek(), stack.isEmpty()

•       Queue<Integer> queue = new LinkedList<>(); — FIFO. queue.offer(), queue.poll(), queue.peek(), queue.isEmpty()

 

String Operations

•       s.length(), s.charAt(i), s.substring(start, end), s.indexOf(ch)

•       s.toCharArray() — converts string to char array, used constantly in character-frequency problems

•       String.valueOf(num) — converts number to string

•       StringBuilder sb = new StringBuilder(); sb.append(); sb.toString(); — use this for string building in loops, not + concatenation

 

Math Class

•       Math.max(a, b), Math.min(a, b) — used in almost every DP problem

•       Math.abs(a) — absolute value

•       Math.sqrt(n) — for prime checking

•       Math.pow(base, exp) — returns double, cast to int if needed

SECTION 4

LeetCode Strategy — How to Actually Solve Problems

 

4.1  The 25-Minute Rule (Non-Negotiable)

When you sit down to solve a problem:

8.    Read the problem twice. Write down the inputs and expected outputs by hand.

9.    Think about the brute force solution first. What is the simplest way, even if slow?

10.  Write the brute force in Java. Get it working first. Don't optimize yet.

11.  Now think: where is the repeated work? Can I cache it? Can I sort first? Can I use a HashMap?

12.  If you are stuck after 25 minutes: look at the solution, understand it fully, close it, and write it from memory.

13.  One week later, attempt the same problem again without looking at your previous solution.

 

4.2  The 3-Layer Understanding

For every problem you solve, you must be able to answer all three:

•       Layer 1 — What: What data structure did I use and why?

•       Layer 2 — How: How does the algorithm work step by step?

•       Layer 3 — Why: Why is this the optimal solution? What makes it better than brute force?

If you cannot answer all three, you have not solved the problem — you have memorized a pattern. Only Layer 3 understanding survives in interviews.

4.4  Problem Patterns to Recognize

The best competitive programmers don't solve new problems — they recognize patterns. These are the 10 patterns that cover 80% of LeetCode problems:

 

Two Pointers

Array is sorted or you need to find a pair. Pointers move toward each other.

Two Sum (sorted), Container With Most Water

 

Sliding Window

Find subarray/substring with constraint. Window expands right, shrinks left.

Longest Substring Without Repeating, Max Sum Subarray

 

Binary Search

Sorted array, or answer lies in a range. Eliminate half the options each step.

Search in Rotated Array, Find Peak Element

 

BFS/DFS

Graph or tree problem. BFS for shortest path, DFS for all paths.

Number of Islands, Word Ladder, Path Sum

 

Dynamic Programming

Optimal substructure — optimal solution uses optimal sub-solutions.

Coin Change, Longest Increasing Subsequence

 

Backtracking

Find all solutions, or determine if any solution exists.

Subsets, Permutations, N-Queens

 

HashMap Frequency

Count occurrences, find duplicates, check if two things are related.

Two Sum, Anagram Check, Top K Frequent

 

Stack Monotonic

Next greater/smaller element. Events that cancel previous events.

Daily Temperatures, Largest Rectangle in Histogram

 

Heap / Top K

K largest/smallest. Dynamic ordering.

Kth Largest, Merge K Sorted Lists

 

Union Find

Group elements, detect cycles, connected components.

Number of Connected Components, Redundant Connection
